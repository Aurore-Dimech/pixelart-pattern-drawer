## Objectif du Bloc 1

Compléter l'application pour qu'elle soit entièrement utilisable en démo :

## Tâches

### T1 — Schéma Prisma : ajout Tag

**Fichier** : `prisma/schema.prisma`  
**Migration** : `npx prisma migrate dev --name add-tags`

Ajouter :
```prisma
model Tag {
  id       String       @id @default(cuid())
  name     String       @unique  // ex: "nature", "espace", "fantasy"
  slug     String       @unique  // kebab-case, généré depuis name
  drawings DrawingTag[]
}

model DrawingTag {
  drawingId String
  tagId     String
  drawing   Drawing @relation(fields: [drawingId], references: [id], onDelete: Cascade)
  tag       Tag     @relation(fields: [tagId], references: [id], onDelete: Cascade)

  @@id([drawingId, tagId])
}
```

Modifier `Drawing` : ajouter `tags DrawingTag[]`

**Gate T1** : `npx prisma studio` affiche les tables Tag et DrawingTag.

### T3 — Routes API manquantes

#### T3.1 — `src/app/api/drawings/[id]/route.ts`
- `GET` : récupère un dessin (ownership OU isPublished)
- `PUT` : met à jour titre + gridData + tags (ownership check)
- `DELETE` : supprime (ownership check)

#### T3.2 — `src/app/api/drawings/[id]/publish/route.ts`
- `POST` : toggle `isPublished` (ownership check)

#### T3.3 — `src/app/api/gallery/route.ts`
- `GET` : liste des dessins publiés
  - Paramètres query : `?search=<titre>&tag=<slug>&page=<n>`
  - Inclut author name, favoriteCount, et si l'utilisateur connecté a mis en favori

#### T3.4 — `src/app/api/favorites/route.ts`
- `GET` : liste les favoris de l'utilisateur connecté
- `POST` : ajoute un favori `{ drawingId }`

#### T3.5 — `src/app/api/favorites/[drawingId]/route.ts`
- `DELETE` : retire le favori

**Gate T3** : curl `GET /api/gallery` retourne une liste, `POST /api/favorites` retourne 201, `DELETE /api/favorites/[id]` retourne 200.

### T5 — Page `/dashboard`

**Fichier** : `src/app/dashboard/page.tsx`  
Protection par auth (redirect si non connecté).

- Liste des dessins de l'utilisateur (carte : aperçu miniature, titre, tags, date, statut publié)
- Bouton "Nouveau dessin" → `/editor`
- Bouton "Éditer" → `/editor/[id]`
- Bouton "Publier / Dépublier" → appel `POST /api/drawings/[id]/publish`
- Bouton "Supprimer" → confirmation inline + appel `DELETE /api/drawings/[id]`

**Aperçu miniature** : `<canvas>` 64×64px rendu côté client depuis `gridData`.

**Gate T5** : Un utilisateur voit ses dessins. Cliquer "Supprimer" retire immédiatement la carte.
